# 小说逻辑审查

## 触发条件

用户要求检查小说逻辑、审查硬伤、检查前后矛盾。

## 审查前置：加载设定文件

**开始审查前必须先读取**：
- `世界观.md` — 战力规则、社会结构、硬性设定
- `角色.md` — 角色属性（年龄/等级/系别/性格）
- `大纲.md` — 章节规划、升级节点、伏笔规划
- `chapter-index.json` — 已写章节摘要和关键事件

不加载设定文件直接审查，会漏掉跨文件矛盾。

## 检查范围（4类）

### 1. 悬空描述
读者看到描述/结论但缺少前置信息支撑，导致读不懂。
- 标志：引用了一个未在上下文中出现过的信息

### 2. 时间线矛盾
- 同一天内事件顺序不对
- 距离/时间估算不合理（步行5分钟走了3公里）
- 冷却时间/倒计时计算错误
- 季节/月份跳变无过渡
- 跨章事件时间冲突

### 3. 设定矛盾
- 前面说了某个规则后面违反了
- 面板数据格式前后不一致
- 角色能力/状态前后矛盾
- 能力表现力与等级不符
- 世界观硬规则被违反无合理解释
- 角色行为前后矛盾

### 4. 信息前后不一致
- 同一事件不同章节描述有出入
- 角色名字/称呼前后不一致
- 数字/编号前后不一致
- 术语/名词混用
- 角色基础信息跨文件矛盾

## 执行步骤

1. 确认小说路径和总章数（用 Glob `chapters/chapter-*.md` 统计，排除 v1-old/ 子目录）
2. 按每批 2 章串行处理
3. 每批处理时：
   - Read 本章 + 前一章正文
   - Read 设定文件（已在步骤1加载）
   - 逐项检查 4 类问题
   - **跨章验证**：对以下关键词做 Grep 全文搜索（`path="{book_dir}/chapters/"`，glob 排除 v1-old/）
     - 本章出现的**角色名**（搜等级/称呼/状态）
     - 本章涉及的**异能名称**（搜能力描写）
     - 本章提到的**数字/编号**（搜同一数值）
     - 本章涉及的**时间表述**（搜"上周""去年""X天"）
   - 发现匹配后读取相关章节片段交叉对比
4. 所有批次完成后，汇总去重，按严重度分级

## 输出格式

```
## 🔴 必须修的硬伤
| # | 章节 | 类型 | 引用原文 | 原因 |

## 🟡 建议修的
| # | 章节 | 类型 | 引用原文 | 原因 |

## ⚠️ 需确认是否真问题
| # | 章节 | 类型 | 引用原文 | 原因 |
```

## 修复原则

修复逻辑漏洞时，修复方案本身不能制造新问题：

1. **保持角色能力一致性**：弱角色不能靠"运气"打败强角色
2. **修漏洞的正确思路**：让弱角色靠**技巧/经验/环境利用**获得短暂优势
3. **交叉引用同步**：改了一个事件的因果描述，Grep 搜索所有章节中的引用并逐一处理
4. **元叙事泄露**：修完后检查是否出现了"章节""集数"等打破第四面墙的词

### 级联编辑

1. 改完主情节后，Grep 搜索关键词全文验证
2. 逐条检查每个引用是否需要同步修改
3. 改完后 Grep 扫旧值关键词，确认零残留

### 用户否决时的处理

连续两次修补同一方案都被否，就该换方向，而不是换参数。
- 方案A被否 → 方案B调整参数 ❌ → 方案C换思路 ✅

## 关键经验

- 不要读 intent/audit 文件——会把规划代号当成正文问题误报
- 角色信息跨章验证要追溯到最早出现的那章
- NPC知道的信息要有来源（"听说的""朋友圈发的"）
- 面板/系统类精度描写要与警告级别匹配
- chapter-index.json 中可能残留已删章节的数据条目，审查报告中需标注
