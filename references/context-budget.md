# 上下文预算分配策略

## 预算总览

| 数据类型 | 预算 | 裁剪策略 |
|---------|------|---------|
| 硬约束（constraints） | ≤300字 | Step 1 生成，语义压缩 |
| 写法规则 | ≤400字 | 只保留标题行+列表行 |
| 大纲（本章） | ≤800字 | Grep 精确提取本章段落 |
| 台账摘要 | ≤1500字 | 按条数限制 |
| 近文参考 | ≤4000字 | 前章2000+500，前2章各500 |
| 世界观 | 全文 | 通常 <25K 字符 |
| 角色 | 全文 | 通常 <5K 字符 |
| 角色状态 | 按需 | 每人最近2条 changes |
| 禁用词 | 全文 | ~2KB，必须完整 |

**总估算**：~30K 字符 ≈ 15K token，远低于 200K token 限制。

## 粗裁剪策略

与原项目精确裁剪不同，纯指令版采用粗裁剪：

1. **世界观 + 角色**：全文保留。如果总字符数 >50K，按段落边界裁剪世界观
2. **写法规则**：只保留以 `#`、`-`、`*`、`❌`、`✅` 开头的行，超过 400 字按段落截断
3. **大纲**：用 Grep 提取包含 `| {N} |` 的行及前后 3 行，超过 800 字截断
4. **台账摘要**：伏笔 ≤10 条（content 截断到 50 字），冲突 ≤6 条（content 截断到 50 字），章节摘要最近 15 条
5. **近文参考**：前一章后半 2000 字 + 末尾 500 字；倒数第二、三章各取末尾 500 字

## 上下文组装顺序

按以下优先级拼接（前面的内容 writer 更容易关注到）：

1. 本章硬约束（constraints）
2. 写作规则
3. 禁用词
4. 世界观
5. 角色
6. 角色当前状态
7. 本章大纲
8. 前一章末尾
9. 最近章节摘要
10. 活跃伏笔
11. 活跃冲突
12. 输出路径

## 关键约束

- 世界观超过安全线（50K 字符）时，用 `## 核心约束（不可违反）` 在顶部单独提取关键规则，再附全文
- 大纲与设定矛盾时，两者同时呈现，让 writer 自行判断
- 面板能力展示需标注当前处于哪一层级，允许展示到什么程度
