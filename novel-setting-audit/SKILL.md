# 小说设定文件交叉审查

对世界观、大纲、角色等设定文件进行系统性交叉审查，找出矛盾、遗漏和逻辑漏洞。

## 审查流程

### 1. 读取所有设定文件

必须同时读取以下文件：
- `世界观.md`
- `大纲.md`
- `角色.md`
- 已完成的最新章节（如有）
- `伏笔台账.json`

### 2. 七维度审查

按以下 7 个维度逐项检查：

**维度1：世界观内部自洽**
- 力量体系是否自洽（分类、克制、等级、跨阶规则）
- 核心设定 Hard Facts 之间是否矛盾
- 面板系统规则是否前后一致

**维度2：大纲 vs 世界观**
- 大纲事件是否违反世界观规则（如跨阶战斗、能力限制）
- 升级节点是否符合世界观机制
- 势力/组织名称是否在两处都有记录

**维度3：角色 vs 世界观**
- 角色异能系别分类是否正确
- 角色等级是否在合理范围
- 角色能力描述是否超出世界观定义

**维度4：大纲 vs 角色**
- 角色在大纲中的行为是否符合人设
- 角色关系发展是否合理
- 配角出场和退场时机

**维度5：伏笔规划**
- 埋设和兑现是否有对应
- 是否有悬而未决的伏笔
- 暗线设定是否与明线矛盾

**维度6：数值一致性**
- 等级比例、人口数字、概率等是否在正文和设定中一致
- 百分比换算是否正确
- 距离、时间等具体数值

**维度7：遗漏设定**
- 写正文时需要但设定文件没覆盖的内容
- 正文已出现但设定未记录的组织/地名/概念
- 角色在大纲中的作用缺少角色卡支撑

### 3. 分级输出

```
## 严重问题（必须修，会导致硬伤）
1. [问题描述] → 文件:行号 → 建议

## 中等问题（建议修，影响质量）
1. [问题描述] → 文件:行号 → 建议

## 轻微问题（可选修）
1. [问题描述] → 文件:行号 → 建议

## 遗漏设定（写正文时需要但没覆盖）
1. [内容]
```

### 4. 修复顺序

严重 > 中等 > 轻微 > 遗漏。修完设定后必须检查已有章节是否需要同步更新。

### 5. 同步清单

修完设定文件后，检查以下文件是否需要同步：
- [ ] 已完成正文中涉及的设定变化
- [ ] 审计报告（audit.md）
- [ ] 伏笔台账（伏笔台账.json）
- [ ] Session Handoff（session-handoff.md）
- [ ] 角色面板等级一览表

## 常见陷阱

1. **面板实体vs虚拟**：面板是脑中虚拟光屏，注意正文不要写成实体
2. **数值换算错误**：百分比×总人口=实际人数，要算账不要拍脑袋
3. **异能系别分类**：涉及规则/空间/时间的能力归特殊系，不是元素系
4. **角色身份一致性**：正文里提到的角色身份必须与角色卡一致
5. **Hard Facts是绝对规则**：注意区分"普通人"和"主角例外"的限定
6. **修改一处要改所有引用**：改了世界观数值，正文、audit、角色卡都要同步
7. **官方评定措辞不要带主观判断**：管理局面板是客观描述能力的
8. **克制链每条都必须是机制对抗**：不是"对方弱"，而是"机制A克制机制B"
9. **社会地位要有数量级支撑**：用总人口×觉醒率×等级比例判断合理性
10. **正文台词不能与世界观数据矛盾**
