# 网文章节润色

## 前置条件

1. 确认用户要润色的章节范围和书籍路径
2. Read 目标章节原文
3. Read 后续章节（ch4+），梳理对目标章节的**依赖关系**
4. 识别**安全区**和**危险区**

## 安全区 vs 危险区

| 类别 | 安全区（随便改） | 危险区（不能动） |
|------|-----------------|-----------------|
| 剧情 | 改写法/节奏/措辞 | 删/改伏笔、调整场景顺序 |
| 设定 | 增加场景细节 | 改面板数据、等级评定、异能描述 |
| 角色 | 对话更口语化 | 改角色行为、性格、立场 |
| 数据 | 压缩info dump | 改数值、格式、计量单位 |
| 钩子 | 加强章末钩子 | 删/改悬念线索 |

**危险区识别方法**：Grep 后续章节（排除 `*-old/` 子目录）中引用的专有名词、面板数据、角色行为，逐一核对。

## 番茄小说前3章专项优化

### 第1章：300字内必须建立冲突
- 砍掉纯场景描写（天气、环境、排队），从动作/对话切入
- 世界观通过角色对话自然渗透，不要 info dump
- 章末钩子要强：用生理反应（呼吸停顿、心跳加速、指尖掐床单）代替内心独白

### 第2章：建立信息差爽感
- 被羞辱/打压场景中，主角不是被动挨骂——他在**观察对手的弱点**
- 读者看到的是"表面被骂，眼睛在捕捉关键信息"
- 走回去的路上主角心不在焉地回应朋友，脑子在消化信息

### 第3章：金手指首秀要有冲击力
- 面板/能力第一次完整展现时，增加三重感官冲击：
  1. 呼吸/心跳的生理冻结
  2. 视野变化（边缘发白、世界缩小）
  3. 余震感（耳鸣、出汗、发凉）
- 章末用意象制造压迫感

## 节奏分析框架

将章节按场景分割（约 300-500 字一段），逐段评分：

| 评分 | 含义 |
|------|------|
| ⭐⭐⭐⭐⭐ | 冲突/爽点/关键信息/强悬念 |
| ⭐⭐⭐ | 有信息推进，但密度一般 |
| ⭐⭐ | 低密度：重复铺垫、纯过渡、流水账 |

**关键指标**：
- 主要冲突/爽点是否在前 200 字内入场
- 连续低密度段落是否超过 400 字（超过=读者可能流失）
- 章节中后段是否有明确的情绪拉升
- 低密度段落是否有功能（推进信息/埋线/建立张力），没有功能的纯水段落必须砍或合并

**黄金3章专项标准**：
- 第1章：300字内建立冲突，不能有超过200字的纯场景铺垫
- 第2章：主要对手/冲突前200字入场，中间低密度不超过300字
- 第3章：金手指首秀在章前1/3处，后半段必须持续拉升

## 写法规则

润色时必须遵守以下规则（完整规则 Read `~/.claude/skills/novel-writer/shared/writing-rules.md`）：
- 对话口语化，标签多样
- 情感通过动作/生理反应间接表达
- 句式长度要有变化
- 不出现英文混入正文（设定内特殊字符除外）
- 面板数据用中文格式（`键：值`，`|` 分隔）

## 禁用词

完整禁用词列表 Read `~/.claude/skills/novel-writer/shared/banned-words.md`。润色时必须确保零触发。

## 执行方式

1. Read 原文 + 安全区/危险区清单 + 写法规则 + 禁用词
2. 执行节奏分析（按场景分段评分）
3. 执行润色（遵守安全区边界）
4. Write 润色后的章节文件
5. 对比字数变化，润色应该是压缩或持平，不应该大幅膨胀
6. 如果修改了正文内容，提醒用户执行 resync
